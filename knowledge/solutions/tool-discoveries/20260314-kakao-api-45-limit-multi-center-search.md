# Insight: 카카오 API pageable_count 45 제한을 다중 중심점 검색으로 우회

## Context
- 시기: 2026-03 카카오맵 맛집 추천 도구(matjip) 크롬 익스텐션 개발
- 상황: 좌표 기반 카테고리 검색(`/v2/local/search/category.json`)으로 주변 음식점을 가져오는 기능 구현 중
- 목표: 반경 500m 내 모든 음식점을 빠짐없이 가져와서 리뷰 분석 후 랭킹

## What We Tried
**접근 1: page 파라미터 증가**
- 카카오 API는 `page: 1~45`, `size: 1~15`를 지원하지만 `meta.pageable_count`가 45로 고정
- 결과: page 4부터 `is_end: true`로 데이터 없음. 최대 45개(3페이지 x 15개)
- 판단: API 자체의 하드 리밋으로 파라미터 조정으로는 우회 불가

**접근 2: radius를 줄여서 여러 번 검색**
- 반경을 작게 잡으면 결과가 줄어 45개 안에 들 수 있지만, 관리 복잡도 증가
- 판단: 반경이 작아지면 경계에 있는 장소가 누락될 수 있음

## What Worked
중심점을 원래 위치 + 4방향(동서남북)으로 오프셋하여 5회 검색, 중복 제거.

**구체적 행동:**
- 원래 반경의 40%만큼 동서남북으로 중심점 이동 (위도 1도 ≈ 111km, 경도 1도 ≈ 88km 기준)
- 각 중심점에서 동일 반경으로 카테고리 검색 (각각 최대 45개)
- place ID 기준 중복 제거 후 원래 중심점 기준 거리 재계산(haversine)

**측정 가능한 결과:**
- 수지구청 500m: 45개 → 135개 (3배)
- 누락되었던 소담촌 수지점(194m) 정상 포함
- API 호출 5배 증가하지만 24시간 캐시로 반복 검색 시 영향 없음

## Why This Works
1. **근본 패턴:** 카카오 API의 45개 제한은 "하나의 검색 요청"에 대한 것. 중심 좌표가 다르면 독립적인 요청으로 각각 45개를 받을 수 있다.
2. **구조적 이유:** 공간 검색은 중심점이 달라지면 결과 정렬이 바뀐다. A 중심에서 46번째였던 장소가 B 중심에서는 10번째일 수 있다.
3. **전제 조건:** 결과에 고유 ID가 있어 중복 제거가 가능해야 하고, 거리를 직접 재계산할 수 있어야 한다.

## Replication Guide
- **적용 조건:** 좌표 기반 반경 검색에 결과 수 제한이 있는 모든 API (카카오, 네이버, Google Places 등)
- **단계별 절차:**
  1. 오프셋 계산: `offset = radius * 0.4`, `dLat = offset / 111000`, `dLng = offset / 88000`
  2. 중심점 5개 생성: [원점, 북, 남, 동, 서]
  3. 각 중심에서 동일 파라미터로 검색
  4. ID 기준 중복 제거, 원래 중심 기준 haversine 거리 재계산
  5. 거리순 정렬
- **주의사항:** 오프셋 비율(0.4)은 경험적 값. 너무 크면 반경 밖 결과가 많아지고, 너무 작으면 겹침이 많아 효과 감소
- **변형:** 9방향(+대각선)으로 확장하면 225개+까지 가능하지만 API 호출 비용 증가. 필요한 커버리지에 따라 조절
