# Insight: 카카오맵 영문 주소를 한글 검색어로 찾으려면 한영 지역명 매핑이 필요하다

## Context
- 시기: 2026-03 카카오맵 맛집 추천 도구(matjip) 크롬 익스텐션 개발
- 상황: 즐겨찾기(4000+개) 검색 기능에서 한글 키워드로 주소/이름/메모를 검색
- 목표: '강남', '수지' 같은 한글 지역명으로 해당 지역의 모든 즐겨찾기를 검색

## What We Tried
**접근 1: 단순 문자열 포함(includes) 검색**
- 검색어를 `place_name`, `address_name`, `road_address_name`, `memo` 필드에 대해 includes 매칭
- 결과: '강남' 검색 시 스터번 등 일부 장소 누락
- 판단: 카카오맵 API가 일부 장소의 주소를 영문으로 반환 — "Gangnam-gu"에는 "강남"이 포함되지 않음

## What Worked
한국 주요 지역명의 한글↔영문 매핑 테이블을 만들어, 검색어가 매핑에 존재하면 영문 변환도 함께 검색.

**구체적 행동:**
- 주요 시/구/동의 한글-영문 매핑 딕셔너리 작성 (강남↔Gangnam, 수지↔Suji, 서초↔Seocho 등 약 50개)
- 검색 시 한글 키워드에 대응하는 영문 키워드를 추출
- 원본 검색 + 영문 변환 검색을 OR 조건으로 결합
- CLI(`fav list -s`)와 크롬 익스텐션 양쪽에 동일 로직 적용

**측정 가능한 결과:**
- '강남' 검색: 매핑 전 누락 있음 → 매핑 후 영문 주소 장소 포함 정상 검색
- 추가 비용: 매핑 테이블 약 50개 항목, 검색 시간 영향 무시 가능

## Why This Works
1. **근본 패턴:** 카카오맵 API는 장소에 따라 주소를 한글 또는 영문으로 반환한다. 이는 API의 데이터 소스가 일관되지 않기 때문이며, 소비자 측에서 정규화해야 한다.
2. **구조적 이유:** 한국 지역명의 로마자 표기는 정부 표준(Revised Romanization)이 있어 1:1 매핑이 가능하다. 매핑 테이블이 작고 정확하므로 유지 비용이 낮다.
3. **전제 조건:** 영문 주소가 표준 로마자 표기를 따라야 한다. 비표준 표기(예: Kangnam vs Gangnam)가 존재하면 매핑에 변형을 추가해야 한다.

## Replication Guide
- **적용 조건:** API가 반환하는 주소 언어가 일관되지 않는 모든 서비스
- **단계별 절차:**
  1. API 응답에서 영문 주소가 포함된 사례 수집
  2. 대상 국가/지역의 로마자 표기 표준 확인 (한국: 국립국어원 로마자 표기법)
  3. 한글↔영문 매핑 딕셔너리 작성 (시, 구, 동 단위)
  4. 검색 함수에서: 검색어 → 매핑 조회 → 원본 + 영문 OR 검색
  5. 클라이언트(익스텐션)와 서버(CLI) 양쪽에 동일 매핑 적용
- **주의사항:** 매핑은 주요 지역만 포함하면 충분. 전국 모든 동/리까지 포함하면 유지 비용 대비 효과가 낮음. 검색 누락 리포트 시 점진적으로 추가
- **변형:** 역방향 매핑(영문→한글)도 추가하면 영문 키워드 검색도 가능. 일본어(구→ku, 시→shi) 등 다른 언어에도 동일 패턴 적용 가능
