# Insight: 세글자 닉네임 가짜 리뷰는 '빈도'가 아닌 '연속성'으로 탐지해야 한다

## Context
- 시기: 2026-03 카카오맵 기반 맛집 추천 도구(matjip) 크롬 익스텐션 개발
- 상황: 가짜 리뷰 의심도 계산 시 "세글자 한글 닉네임 + 5점" 리뷰 비율을 하나의 규칙으로 사용 중
- 목표: 세글자 닉네임 규칙의 false positive를 줄여 탐지 정확도 향상

## What We Tried
**접근 1: 5점 리뷰 중 세글자 닉네임 비율 계산**
- 5점 리뷰만 추출 → 그 중 세글자 한글 닉네임 비율 측정 → 비율이 높으면 의심
- 결과: 진짜 사용자 중에도 세글자 이름이 많아 false positive 빈번
- 판단: 한국인 이름 특성상 세글자(김민수, 이지은 등)가 자연적으로 많으므로 비율만으로는 불충분

**접근 2: 전체 리뷰 타임라인에서 연속 패턴 탐지**
- 전체 리뷰를 날짜순 정렬 → 세글자+5점이 연속 3건 이상 나오는 구간 탐지
- 결과: false positive 대폭 감소, 실제 알바 리뷰 패턴과 정확히 일치

## What Worked
전체 리뷰 타임라인(별점 무관)을 날짜순 정렬하고, "5점 + 세글자 한글 닉네임"이 연속으로 3건 이상 나오는 구간을 의심 패턴으로 플래그.

**구체적 행동:**
- 전체 리뷰를 날짜순 정렬 (5점 필터링 없이 모든 리뷰 포함)
- 각 리뷰를 순회하며 `rating >= 5 && isThreeCharKorean(nickname)` 조건 체크
- 연속 카운터 유지: 조건 충족 시 +1, 미충족 시 0으로 리셋
- 연속 카운터가 3 이상이면 해당 구간을 의심 플래그로 기록
- 최종 의심 점수에 연속 구간 수와 길이를 반영

**측정 가능한 결과:**
- 기존 비율 기반: false positive 多 (자연적으로 세글자 이름이 30%+ 되는 가게)
- 연속 패턴 기반: 실제 알바 투입 패턴만 정확히 탐지
- 핵심 차이: 2점, 3점 리뷰가 사이에 끼면 연속성이 끊겨 자연스러운 분포와 구분 가능

## Why This Works
1. **근본 패턴:** 리뷰 알바 업체는 비용 효율을 위해 같은 시기에 여러 계정을 일괄 투입한다. 이 계정들은 대부분 세글자 한글 이름을 사용하고 5점만 부여하므로, 타임라인에서 "연속된 세글자+5점 블록"이 형성된다.
2. **구조적 이유:** 일반 사용자의 리뷰는 시간적으로 분산되고, 사이에 다양한 별점(1~4점)과 다양한 닉네임 형식이 섞인다. 연속성은 이 자연적 분포와 조직적 투입을 구분하는 가장 강력한 축이다.
3. **전제 조건:** 리뷰 목록을 날짜순으로 정렬할 수 있어야 하고, 닉네임과 별점 정보가 함께 제공되어야 한다.

## Replication Guide
- **적용 조건:** 리뷰가 시간순으로 나열되고 닉네임+별점을 확인할 수 있는 모든 리뷰 플랫폼
- **단계별 절차:**
  1. 전체 리뷰를 날짜순 정렬 (최신→오래된 순 또는 반대)
  2. 순회하며 `(별점 >= 5) && (닉네임이 세글자 한글)` 조건 체크
  3. 연속 카운터: 조건 충족 시 +1, 미충족 시 0 리셋
  4. 연속 카운터 >= 3이면 의심 플래그
  5. 플래그된 구간 수에 따라 의심 점수에 가중치 부여
- **주의사항:** "세글자"의 정의는 플랫폼별 닉네임 정책에 따라 조정. 카카오맵은 한글 2~4자 닉네임이 일반적이므로 정확히 3자(한글)를 기준으로 함
- **변형:** 연속 기준을 2로 낮추면 recall 증가/precision 감소, 4로 높이면 반대. 리뷰 수가 적은 가게(20개 미만)에서는 2로, 많은 가게에서는 4로 조정 가능
